# Railway MongoDB Environment Variables - Explained

**Date**: 2026-07-16
**Context**: Understanding the three MongoDB variables in Railway

---

## The Three MongoDB Variables

When Railway provisions a MongoDB service, it automatically creates multiple environment variables. Here's what each one means:

### 1. **MONGO_URL** (Railway Internal - Private Network)

**Format**:
```
mongodb://mongo:<password>@mongodb.railway.internal:27017
```

**What it is**:
- Railway's **internal private network** URL
- Uses Railway's private DNS (`mongodb.railway.internal`)
- Only accessible within Railway's internal network
- Intended for service-to-service communication

**When to use**:
- ❌ **DO NOT USE** - Known to fail with DNS resolution errors
- Railway's internal DNS doesn't resolve reliably
- Error: `getaddrinfo ENOTFOUND mongodb.railway.internal`

**Example**:
```
mongodb://mongo:LegGfRDdP...@mongodb.railway.internal:27017
```

---

### 2. **MONGO_PUBLIC_URL** (Railway TCP Proxy - Public)

**Format**:
```
mongodb://mongo:<password>@metro.proxy.rlwy.net:<port>
```

**What it is**:
- Railway's **TCP proxy** URL
- Routes through Railway's public proxy infrastructure
- Accessible from anywhere (within Railway and externally)
- Uses stable DNS that resolves correctly
- Port is dynamically assigned by Railway

**When to use**:
- ✅ **RECOMMENDED** for all Flora services
- Reliable, works consistently
- Used by flora-mcp-server (working)
- Stable across deployments

**Example**:
```
mongodb://mongo:LegGfRDdPGDxZgqDGbqjFJWlWASmCGNB@metro.proxy.rlwy.net:59998
```

---

### 3. **MONGODB_URI** (Custom Variable - Your Choice)

**Format**:
```
mongodb://<user>:<password>@<host>:<port>/<database>
```

**What it is**:
- **Custom environment variable** you set manually
- Not auto-generated by Railway
- Used by your application code (e.g., `process.env.MONGODB_URI`)
- Can point to any MongoDB instance (Railway, Atlas, self-hosted, etc.)

**When to use**:
- ✅ **REQUIRED** - Your application code expects this variable
- Set it to the value of `MONGO_PUBLIC_URL` + database name
- Allows you to control which database to use

**Example**:
```
mongodb://mongo:LegGfRDdPGDxZgqDGbqjFJWlWASmCGNB@metro.proxy.rlwy.net:59998/venturestudio
```

---

## Why Three Variables Exist

Railway creates multiple variables to give you flexibility:

| Variable | Purpose | Railway Auto-Creates | Works Reliably |
|----------|---------|---------------------|----------------|
| `MONGO_URL` | Internal private network | ✅ Yes | ❌ No (DNS fails) |
| `MONGO_PUBLIC_URL` | Public TCP proxy | ✅ Yes | ✅ Yes |
| `MONGODB_URI` | Your custom config | ❌ No (you set it) | ✅ Yes (if set correctly) |

**Railway's intent**:
- `MONGO_URL` → Use for internal service communication (theory)
- `MONGO_PUBLIC_URL` → Use for external connections (reality: use for everything)
- `MONGODB_URI` → You define what MongoDB to use

**Reality in practice**:
- `MONGO_URL` → ❌ Doesn't work (DNS issues)
- `MONGO_PUBLIC_URL` → ✅ Works perfectly
- `MONGODB_URI` → ✅ Set it to `MONGO_PUBLIC_URL` + `/database_name`

## Which Variable Should Flora DocAssemble Service Use?

### Your Application Code Expects:

**File**: `src/config/database.js`
```javascript
const mongoURI = process.env.MONGODB_URI || 'mongodb://localhost:27017/flora-docassemble';
```

Your code looks for **`MONGODB_URI`** (not `MONGO_URL` or `MONGO_PUBLIC_URL`).

### What You Should Set:

Set `MONGODB_URI` to the **TCP proxy URL** with your database name:

```bash
MONGODB_URI=mongodb://mongo:LegGfRDdPGDxZgqDGbqjFJWlWASmCGNB@metro.proxy.rlwy.net:59998/venturestudio
```

This is essentially: `MONGO_PUBLIC_URL` + `/venturestudio`

## How to Reference Railway Variables

### Option A: Set Custom Variable (Recommended)

**In Railway Dashboard**:
```
Name: MONGODB_URI
Value: mongodb://mongo:LegGfRDdPGDxZgqDGbqjFJWlWASmCGNB@metro.proxy.rlwy.net:59998/venturestudio
```

**Pros**:
- Explicit and clear
- You control the database name
- Easy to change/debug
- Works consistently

### Option B: Reference Railway Variable (Advanced)

**Using Railway variable reference syntax**:
```
MONGODB_URI=${{MONGO_PUBLIC_URL}}/venturestudio
```

**Pros**:
- Automatically updates if Railway changes the URL
- DRY (don't repeat yourself)

**Cons**:
- Less explicit
- Harder to debug
- May not append database name correctly
- Railway variable syntax can be quirky

### Option C: Use Railway Variable Directly (Not Recommended)

**Change your code to**:
```javascript
const mongoURI = process.env.MONGO_PUBLIC_URL + '/venturestudio';
```

**Cons**:
- Requires code changes
- Ties code to Railway-specific variables
- Less portable
- Not following existing pattern

## Known Railway MongoDB Issues

### Issue 1: Internal DNS Failure

**Symptom**:
```
MongoServerError: getaddrinfo ENOTFOUND mongodb.railway.internal
```

**Cause**: Railway's private network DNS doesn't resolve

**Solution**: Use `MONGO_PUBLIC_URL` instead of `MONGO_URL`

### Issue 2: Service Reference Variables

**Symptom**:
```bash
MONGODB_URI=${{MongoDB-original.MONGO_URL}}  # Points to broken internal DNS
```

**Cause**: Railway variable references may resolve to internal URLs

**Solution**: Set explicit TCP proxy URL

### Issue 3: Missing Database Name

**Symptom**:
```
MongoServerError: Authentication failed
```

**Cause**: Connection string missing `/database_name`

**Solution**: Always include database name in URL:
```
mongodb://...@host:port/venturestudio  # ✅ With database
mongodb://...@host:port                 # ❌ Without database
```
